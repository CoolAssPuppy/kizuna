# Tech Debt Audit — kizuna

Generated: 2026-05-01

## Executive summary

- **Critical: 1.** `transport_requests.pickup_datetime` vs `transport_vehicles.pickup_at` are the same concept under two names; the manifest report and edge-function triggers all reach for `pickup_datetime`, the new tool reaches for `pickup_at`. One rename hardens reads / writes / triggers in one go.
- **High: 6.** 8 registration sections fetch data via `useEffect` instead of TanStack Query — violates the documented `no-useEffect` rule, breaks realtime invalidation, and leaks rendered state on unmount. Stub HiBob directory is duplicated across three modules with no shared source. Two paths to delete an event (`deleteEvent` + `deleteEventCascade`) — only the latter has the admin-guard RPC. `as unknown as <Shape>[]` casts in five files erase Supabase-generated types. `deno-lint-ignore-file no-explicit-any` blanket-disabled across 5 edge functions. Knip flags ~70 type exports as unused — most are real.
- **Medium: 11.** God-file candidates (`GroundTransportToolScreen.tsx` 611 LOC, `RoomAssignmentToolScreen.tsx` 533 LOC). 5 admin tools with `useQuery` + `useMutation` have zero component tests. Stale `home.jetLag.tips.*` i18n keys still in `common.json` after the random-tip refactor. Bash conditional `npm scripts` (`if command -v doppler...`) silently mis-forward args (already burned us in CI). `JetLagFighter.tsx` re-exports `JetLagDirection` / `JetLagSeverity` types nobody imports.
- **Low: 9.** Stale docstrings on `JetLagFighter`, `DevSignInShortcuts`. Knip + depcheck noise in config. Idle `world-atlas` devDep used only at install time. Reference to `tasks/lessons.md` in README without keeping the file in sync. PWA precache lists 65 entries — hand-curating those keeps drifting.
- **Architectural decay**: low. No circular deps (madge clean). Layering is consistent — features → lib → types. The hot zones are the admin tools (recent feature pushes), and most of the debt clusters in those four screens.
- **Security hygiene**: clean for the launch posture. RLS enabled programmatically across every public.\* table. `npm audit` 0 vulnerabilities. No string-concat SQL anywhere. The one reachable `: any` (`userScopedRepository.ts`) is annotated and exposes a fully-typed surface.

## Architectural mental model

Kizuna is a single-page Vite/React app fronting Supabase. The SPA layer is feature-sliced under `src/features/<domain>/`, each feature owning its components, hooks, API calls (via `@supabase/supabase-js`), and tests. Cross-cutting code lives in `src/lib/` (Supabase client singleton, env, i18n, integrations). Types are generated from the live database (`supabase gen types`) and consumed everywhere via `src/types/database.types.ts`.

Backend is declarative SQL in `supabase/schemas/*.sql` (00-prefixed for ordering), applied locally via `scripts/db-apply.sh`. RLS is enabled in a single `do $$` block across every public table; policies live alongside in `90_rls.sql`. Edge functions run in Deno under `supabase/functions/`. The PWA boots even without `VITE_SUPABASE_URL` (placeholder fallback in `env.ts`); the welcome / 404 screens are intentionally backend-free.

State is TanStack Query for server data plus per-component `useState` for UI. Realtime is wrapped in `src/lib/useRealtimeInvalidation.ts` so screens just declare `(table, queryKey)` pairs and refetch on push. There is one documented exception to the no-useEffect rule (registration sections use `useEffect` to hydrate per-section state on mount); the audit treats that exception as debt.

## Top 5 — fix these first

1. **F001 — Rename `pickup_datetime` to `pickup_at`.** One column rename in `supabase/schemas/50_logistics.sql`, then `grep -r pickup_datetime` and update the trigger body, the manifest report, the test fixtures. The two-name drift is a footgun — I almost shipped a wrong query during the Ground Transport Tool work, caught only by a typecheck after the fact.

2. **F002 — Replace the eight registration `useEffect` fetches with `useQuery`.** Each section is structurally identical:

   ```ts
   // BEFORE
   const [values, setValues] = useState(EMPTY);
   useEffect(() => {
     load(getSupabaseClient(), userId).then(setValues);
   }, [userId]);

   // AFTER
   const { data: values = EMPTY } = useQuery({
     queryKey: ['section', userId],
     queryFn: () => load(getSupabaseClient(), userId),
   });
   ```

   The mutation half already lives in `useSectionSubmit` and uses TanStack — the read half just hasn't migrated yet.

3. **F004 — Collapse the two delete-event paths.** Drop `deleteEvent`, rename `deleteEventCascade` to `deleteEvent`, update the one caller in `EventEditScreen`. Removes the silent-failure path for non-admins.

4. **F003 — Stop reaching for `as unknown as`.** Define one `JoinedFlight = QueryData<typeof flightsQuery>` near each query and let typegen do the work. Five files; an hour of work; future typegen regens become useful again.

5. **F008 — Move the doppler shell into `scripts/with-doppler.sh`.** The `if command ... fi` inline pattern already burned us in CI today (the `fi --project=chromium` syntax error). Six scripts; one shared shim with `"$@"` propagation.

## Things that look bad but are actually fine

- **`scripts/reset-remote-db.sh` includes a hardcoded local db password fallback (`postgres`).** That's the canonical local Supabase password. Removing the fallback would just force the user to re-set it everywhere.
- **`src/lib/env.ts` falls back to placeholder strings rather than throwing.** This is the fix for the e2e / deploy-preview failures — unbiased read says "throw at boot is purer," but it's been net-negative every time it fired. Leaving the placeholder fallback in place.
- **`useRealtimeInvalidation` has an `eslint-disable-next-line react-hooks/exhaustive-deps`.** The custom hook deliberately keeps the binding list out of deps because it stringifies them into a fingerprint instead. Disabling the rule here is correct.
- **`pickJetLagTip` and `pickIcebreakerTarget` use `Math.abs(seed) % len`.** Looks like a dumb hash but it's deliberate — the seed is `Date.now()` and we want determinism per render pass with cheap rerolls.
- **`JetLagFighter` returns `null` when offset is 'none'.** Reads like accidentally hidden UI but matches the spec — the card is jet-lag-specific and shouldn't render for someone already in the venue tz.
- **`supabase/seed.sql` marks the canonical guest as `payment_status = 'paid'`.** Looks like seed pollution but it's there to satisfy the `guard_guest_profile_completion` trigger that I just landed; without it, applying the seed throws.
- **`additional_guests` is private-by-default with NO `attendee_profiles` row.** Looks like a missing-data bug; it's the "minor profiles are private" feature spec.
- **`src/lib/integrations/hibob.ts` exports a `team: 'Database'` field on the stub even though the live API doesn't surface team.** Stub-only, fed into the reconcile tests.
