## Architectural mental model

Three runtimes share one Postgres database:

- **SPA** (Vite/React/TS) reads through Supabase JS SDK with RLS as the only authorization layer.
- **Edge functions** (Deno) hold the integration glue — Stripe, OpenAI, HiBob, Resend, Slack — and the privileged operations (signed invitations, share-report tokens).
- **Postgres** owns invariants: triggers maintain `registrations.completion_pct` and materialise `itinerary_items`; RLS gates every table; pgcrypto encrypts passport numbers; SECURITY DEFINER RPCs cover narrow privileged paths.

Each runtime is independently deployable. Constants and integration types are duplicated between `src/lib` and `supabase/functions/_shared` because the Deno runtime cannot import from the SPA's path alias. This is intentional duplication — flagged in the README and lessons.

## Top 5 if-you-fix-nothing-else

1. **Stripe edge-function hardening** (#18). Idempotency keys on create-stripe-checkout (use `crypto.randomUUID()` for the `Idempotency-Key` header), structured logs (one JSON line per request with correlation id), role-based unit tests for every edge function. Production-launch blocker.
2. **`messages.channel_id` FK migration** (#17). Add `channel_id uuid references public.channels(id)`, backfill from slug, drop the text column. Eliminates the entire class of "what does this slug mean" bugs.
3. **IANA timezone validation** (#16). Trigger on `events`, `flights`, `transport_requests`, etc. that calls `pg_timezone_names`. One function, applied to ~5 columns.
4. **E2E coverage for community + channel** (#19). One Playwright spec that signs in, opens /community, clicks a channel, sends a message, asserts it appears. Catches the realtime + RLS interaction in one shot.
5. **README + scripts: doppler in CI** — currently CI uses placeholder env vars. Once Doppler service tokens land in GitHub secrets, switch CI to `doppler run --token $DOPPLER_TOKEN -- npm run X`.

## Things that look bad but are actually fine

- **`supabase/functions/_shared/*` reported as unused by Knip**. Knip parses TypeScript through Node module resolution; Deno edge functions are compiled by `supabase functions deploy` and never imported into Node. Adding `supabase/functions/**` to the Knip ignore list is correct.
- **24 unused exported types** flagged by Knip. Most are interface declarations next to their implementing functions (e.g. `ChannelRow`, `CommunityProfileRow`). They document the row shape for callers and the cost of un-exporting them is brittleness in tests + lint warnings about `prefer-export`. Keep.
- **Constants duplicated between `src/lib/constants.ts` and `supabase/functions/_shared/constants.ts`**. The Deno edge runtime cannot import from `src/`. The duplication is intentional and called out in the file's docstring.
- **`@typescript-eslint/eslint-plugin`, `@typescript-eslint/parser`, `eslint-plugin-react`** flagged as unused devDeps. They're transitively required by `typescript-eslint` (the v8 meta-package) and consumed via the flat config. Removing them breaks lint.
- **`autoprefixer`, `postcss`, `prettier-plugin-tailwindcss`** flagged. Required by tooling configs (`postcss.config.cjs`, `prettier.config.cjs`). Knip doesn't model these.
- **`world-atlas`** flagged. Used at install time to copy `countries-110m.json` into `public/`. Documented in the WorldMap source comment.
- **`@vitest/coverage-v8`** flagged. Used by the `test:coverage` npm script.
- **DB schema files >500 LOC** (#21, #22). Splitting RLS or the trigger file has more cost than benefit: every policy belongs in one auditable place; every trigger maintains a single invariant and reads the others to know it's the source of truth. Splitting would make a security review harder.
